🔑 Phase 1: Glyph Authentication Analysis Glyph OAuth

Glyph is Otherside's authentication hub — it handles login, wallet integration, and fiat onramping. Understanding Glyph's OAuth flow is critical for building tools that interact with the native client.

Glyph Architecture: Glyph (useglyph.io) acts as an OAuth2 provider for Otherside. It bridges wallet-based auth (MetaMask, WalletConnect) with traditional OAuth tokens. The flow: Wallet Signature → Glyph OIDC → Firebase Custom Token → Otherside Session.

🌐 Phase 3: Morpheus Network Analysis Morpheus/SpatialOS

Morpheus is MSquared's networking layer built on top of Improbable's SpatialOS. It handles entity replication, RPCs, and world state synchronization across thousands of concurrent players.

Key Insight: Morpheus uses a proprietary protocol over WebSocket. The native client connects to Morpheus workers via wss://*.msquared.io/* endpoints. Understanding this protocol is essential for building tools that interact with the game world.

Client Memory/Process Analysis Guide Advanced

For local analysis of the running client process, use these tools and techniques:

1
Process Discovery
Find Otherside.exe process and its memory layout. Use Process Monitor to track file/registry/network access.
2
Config Extraction
Extract .ini and .json config files from the game directory. Look for API keys, endpoints, and auth tokens in config files and logs.
3
Network Capture
Use Wireshark/Fiddler to capture WebSocket traffic. Apply TLS interception for HTTPS. Identify Morpheus protocol structure.
4
DLL Injection / Hooking
Inject into the Unreal Engine process to hook networking functions. Intercept calls to FWebSocket::Send/Recv for protocol analysis.
5
Token Extraction
Search process memory for JWT tokens, session cookies, and WebSocket URLs. Look for Firebase ID tokens and refresh tokens in memory.

📺 Phase 5: Pixel Streaming Security WebRTC

Otherside supports browser-based play via Google Cloud Immersive Stream for XR (Pixel Streaming). This uses WebRTC for video delivery and a signaling server for input relay. The attack surface includes WebRTC, signaling, and the input channel.

How Pixel Streaming Works: Unreal Engine runs on a cloud GPU → Video is encoded and streamed via WebRTC → User input (mouse, keyboard, controller) is sent back via a WebSocket data channel → The server processes input and updates the game state. Requires Google Chrome and 25+ Mbps.
