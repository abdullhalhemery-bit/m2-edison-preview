🔑 Phase 1: Glyph Authentication Analysis Glyph OAuth

🌐 Phase 3: Morpheus Network Analysis Morpheus/SpatialOS

Morpheus is MSquared's networking layer built on top of Improbable's SpatialOS. It handles entity replication, RPCs, and world state synchronization across thousands of concurrent players.

Key Insight: Morpheus uses a proprietary protocol over WebSocket. The native client connects to Morpheus workers via wss://*.msquared.io/* endpoints. Understanding this protocol is essential for building tools that interact with the game world.

🖥️ Direct Game Connection Local Proxy

The Otherside desktop client communicates via WebSocket to MSquared infrastructure. When the game is running, it holds active WebSocket connections. We can intercept this traffic.

How it works: Since the browser can't directly connect to localhost game processes, a local Node.js proxy bridges the gap. The proxy listens on a local port, accepts dashboard connections, forwards commands to MSquared endpoints, and captures/relays responses including tokens from the game's WebSocket traffic.
🔴 Disconnected
[LOCAL-PROXY] Set up the local proxy on your machine first (see script below), then connect.

📋 Node.js Local Proxy Script (save as proxy.js and run with node proxy.js)

const WebSocket = require('ws'); const http = require('http'); const PORT = 8765; const MSQUARED_WS = 'wss://gateway.msquared.io/v1/connect'; const ODK_API = 'https://odk.otherside.xyz/api/v0'; // Track captured tokens const capturedTokens = []; const server = http.createServer((req, res) => { res.setHeader('Access-Control-Allow-Origin', '*'); res.setHeader('Content-Type', 'application/json'); if (req.url === '/status') { res.end(JSON.stringify({ status: 'running', capturedTokens: capturedTokens.length, uptime: process.uptime() })); } else if (req.url === '/tokens') { res.end(JSON.stringify(capturedTokens)); } else { res.end(JSON.stringify({ error: 'Unknown endpoint' })); } }); const wss = new WebSocket.Server({ server }); wss.on('connection', (dashboardWs) => { console.log('[PROXY] Dashboard connected'); // Connect to MSquared infrastructure const msquaredWs = new WebSocket(MSQUARED_WS); let authCaptured = false; msquaredWs.on('open', () => { console.log('[PROXY] Connected to MSquared'); dashboardWs.send(JSON.stringify({ type: 'proxy_status', status: 'connected_to_msquared' })); }); msquaredWs.on('message', (data) => { const msg = data.toString(); // Capture any tokens in the traffic const tokenMatch = msg.match(/eyJ[A-Za-z0-9-_]+\.[A-Za-z0-9-_]+\.[A-Za-z0-9-_]+/); if (tokenMatch && !authCaptured) { capturedTokens.push({ token: tokenMatch[0], timestamp: new Date().toISOString(), source: 'msquared_ws' }); authCaptured = true; console.log('[PROXY] Token captured!'); } // Forward to dashboard if (dashboardWs.readyState === WebSocket.OPEN) { dashboardWs.send(JSON.stringify({ type: 'server_message', data: msg, direction: 'IN', timestamp: Date.now() })); } }); // Handle messages from dashboard dashboardWs.on('message', (data) => { try { const cmd = JSON.parse(data.toString()); if (cmd.type === 'forward' && msquaredWs.readyState === WebSocket.OPEN) { msquaredWs.send(cmd.payload); } else if (cmd.type === 'get_tokens') { dashboardWs.send(JSON.stringify({ type: 'tokens', tokens: capturedTokens })); } } catch (e) { console.error('[PROXY] Parse error:', e.message); } }); dashboardWs.on('close', () => { console.log('[PROXY] Dashboard disconnected'); msquaredWs.close(); }); }); server.listen(PORT, () => { console.log(`[PROXY] Listening on ws://localhost:${PORT}`); console.log(`[PROXY] Status: http://localhost:${PORT}/status`); console.log(`[PROXY] Tokens: http://localhost:${PORT}/tokens`); });

Client Memory/Process Analysis Guide Advanced

For local analysis of the running client process, use these tools and techniques:

1
Process Discovery
Find Otherside.exe process and its memory layout. Use Process Monitor to track file/registry/network access.
2
Config Extraction
Extract .ini and .json config files from the game directory. Look for API keys, endpoints, and auth tokens in config files and logs.
3
Network Capture
Use Wireshark/Fiddler to capture WebSocket traffic. Apply TLS interception for HTTPS. Identify Morpheus protocol structure.
4
DLL Injection / Hooking
Inject into the Unreal Engine process to hook networking functions. Intercept calls to FWebSocket::Send/Recv for protocol analysis.
5
Token Extraction
Search process memory for JWT tokens, session cookies, and WebSocket URLs. Look for Firebase ID tokens and refresh tokens in memory.

📺 Phase 5: Pixel Streaming Security WebRTC

Otherside supports browser-based play via Google Cloud Immersive Stream for XR (Pixel Streaming). This uses WebRTC for video delivery and a signaling server for input relay. The attack surface includes WebRTC, signaling, and the input channel.

How Pixel Streaming Works: Unreal Engine runs on a cloud GPU → Video is encoded and streamed via WebRTC → User input (mouse, keyboard, controller) is sent back via a WebSocket data channel → The server processes input and updates the game state. Requires Google Chrome and 25+ Mbps.
